Detailed Description

Operation - abstraction which represent Evm instruction.

Author
Lukasz Iwanecki
Date
05.2018 IOperation interface represents Evm bytecode instruction. The operation interface has execute() method which may be called by an application. Evm operation arguments usually requires access to registers, thus the execution must be performed within a thread context. Each operation needs arguments which are represented by IArgument interface. The operations are capable of printing themselvs. The opcode string is passed in constructor and the application may call trace() function to get string with instruction opcode, printable list of arguments and values of the arguments. Abstraction capable of making the Operations is IOperationFactory
