Detailed Description

Argument interface.

Author
Lukasz Iwanecki
Date
05.2018 IArgument interface represents Evm instruction argument. Arguments are passes to IOperation class. Evm sepcification intriduces a few argument types: address, constant, register, BYTE, WORDm DWORD and QWORD. Each of them can be implemeted as IArgument. Argument values are always 64bit unsigned numbers. It is usually true but there is one exception - address argument is 32-bit. It is up caller class to cast that value to 32-bit, but fortunatly the caller is awared of argument types it uses. The IArgument interface has two classes responsible of printing the arguemnt. label() function prints readable form of the argument. printValue() returns label as well as value of the parameter in runtime
